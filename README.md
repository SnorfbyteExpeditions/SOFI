# 🎮 Game Concept – *The Secret of Fire Island*

## 🧠 Core Idea

Build a **Monkey Island–inspired point-and-click adventure game** with a humorous and modern twist during Wapice's Leap of Fate hackathon on 28.5.2026.

The player controls:

> **Johan Devsson** – an aspiring software developer with zero real-world experience but enormous confidence.

### 🎯 Main Goal

* Become a real software developer
* Defeat the terrifying **HR recruiter (LeRecruiter)**
* Rescue your beloved **“Perfect Project Idea”** (a symbolic, glowing concept representing success)

***

## 🏝️ Setting: Fire Island (Palosaari, Vaasa)

The game takes place in a stylized, semi-fictional version of Palosaari with multiple locations:

* **Technobothnia Campus** → “The Academy of Coding”
* **Student Village** → chaotic hub with side quests
* **Grill Kiosks** → healing items + quirky NPCs
* **Harbor & Ferry** → travel to “Remote Work Island”
* **Abandoned Office Building** → haunted legacy systems
* **Sauna Complex** → key story decisions
* **Frozen shoreline** → environmental puzzle area

***

## 🧑‍🤝‍🧑 Key Characters

* **Johan Devsson** – optimistic but clueless hero
* **Tewo the Mentor** – gives cryptic advice
* **The Rubber Duck** – helpful debugging companion
* **Freelancer NPCs** – pay in “exposure”
* **LeRecruiter** – the main antagonist

***

## 🎮 Gameplay

### Core mechanics:

* Point-and-click interaction
* Character walks around between scenes
* Dialogue-driven progression
* Inventory-based puzzles

### Unique twist:

* **AI-generated dialogue**
  * Plot and logic stay fixed
  * Dialogue varies each playthrough
  * Humor-driven interactions

***

## 🛠️ Tech Stack

I plan to build this as a **web-based game** using:

* **HTML** → structure
* **CSS** → visuals/UI
* **JavaScript** → logic/gameplay

This allows:

* Easy sharing (via link)
* No installation required
* Fast iteration during hackathon

***

## ⚡ Hackathon Scope (6–8 hours)

### 🎯 Goal: Build a vertical slice (NOT full game)

#### ✅ Include:

* 1–2 scenes (e.g., kiosk + campus)
* Player movement (click-to-walk)
* 1 NPC interaction
* 1 simple puzzle chain
* Basic inventory system
* Dialogue system (with optional AI variation)

#### ❌ Avoid:

* Large world
* Complex animation
* Advanced systems (save/load, physics, etc.)

***

## 🧠 Key Strategy

### ✅ 1. Keep it small

Focus on:

> One polished, playable experience (5–10 minutes)

***

### ✅ 2. Fake complexity

* Simple logic behind the scenes
* Rich, funny dialogue creates depth

***

### ✅ 3. Layer development

Build in order:

1. Movement (click-to-walk)
2. Scene switching
3. Interaction system
4. Dialogue
5. Inventory
6. Puzzle

***

### ✅ 4. Use AI effectively

Use AI for:

* Dialogue generation
* Code snippets (small pieces only)
* Asset ideas

Avoid:

* Generating entire systems at once

***

### ✅ 5. Prioritize humor

The game’s strength is:

* Relatable developer jokes
* Satire of tech culture
* Funny interactions

***
